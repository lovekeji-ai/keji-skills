# fogg-habit

> 基于 BJ Fogg《福格行为模型 / Tiny Habits》的对话式习惯教练。用 **B=MAP** 帮你诊断为什么坚持不下来、从愿望设计新习惯、戒除坏习惯、把庆祝设计进去、帮家人/团队做行为改变。

这不是一个 to-do 应用，也不写代码、不落盘。它是一个**能被 Agent 加载的教练脚本**——你说一句"我总是坚持不了跑步"，它按福格的行为设计流程一步步问、一步步引，最后给你一份能直接用的"微行为配方"。

## 它解决什么问题

普通的"养成习惯"建议大多在打鸡血——多喝水、每天早起、坚持就是胜利。福格的核心洞察是：

> **行为 = 动机 × 能力 × 提示**，三者同时出现且越过"行动线"才会发生。缺一个，行为就不发生。

所以真正该做的不是打鸡血，而是：

- **把行为砍到 30 秒内**（微行为），让它几乎不需要动机
- **绑一个已有的锚点**（"在我〔某个已存在的动作〕之后，我会…"）
- **做完立刻庆祝**——情绪，而不是重复次数，才是习惯扎根的原因
- **坚持不下来 = 设计问题，不是意志力问题**——回去改设计，不自责

这个 Skill 把上面这套流程做成了一个**纯对话教练**，一次只问一两个问题，用口语、有温度，不学术腔。

## 五种模式

Agent 会先判断你想做哪件事，再走对应剧本：

| 你说 | 走哪个模式 |
|---|---|
| "我为什么坚持不下来 X" / "这个习惯断了" | **诊断排错** |
| "帮我养成 X" / "我想变得更…该怎么开始" | **行为设计**（愿望→行为群→焦点地图→微行为→锚点→庆祝） |
| "帮我戒掉 X" / "怎么改掉坏习惯" | **戒除坏习惯** |
| "我做完了不知道怎么奖励" / "庆祝那块没搞懂" | **庆祝设计** |
| "怎么帮我老婆/孩子/团队养成…" | **帮别人改变** |

诊断和设计常常会串起来——诊断完发现是设计问题，就自然接到行为设计模式去重做一次。

## 触发方式

```text
/fogg-habit
我为什么坚持不下来早起
帮我养成用牙线的习惯
帮我戒掉睡前刷手机
Tiny Habits 帮我看看这个
```

## 一次对话大概长什么样

- Agent 一次只抛一两个问题，等你答完再往下走
- 全程不写文件、不落盘
- 收尾时会主动问一句："要不要把这次聊出来的东西整理成一份小结？"
  - 要 → 输出一段清晰的文字总结（愿望 / 微行为 / 锚点配方 / 庆祝动作 / 诊断要点）
  - 是否落成文件、放在哪里，都由你决定，它不会自作主张
