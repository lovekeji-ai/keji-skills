# 行为设计模式：从愿望到习惯

用于用户想**从头养成一个新习惯**，或想把模糊愿望（"我想更健康/更专注/更会写作"）变成能落地的习惯链路。

这是福格"行为设计"的完整流程。**一步一步走，每步只问一两个问题**，别把整张流程一次倒给用户。

## 目录
1. 明确愿望（Aspiration）
2. 探索行为群（Magic Wand / Swarm of Behaviors）
3. 焦点地图筛黄金行为（Focus Mapping → Golden Behaviors）
4. 从微小起步（Start Tiny）
5. 找锚点（Anchor / After-I-will 配方）
6. 设计庆祝（Celebrate）
7. 收尾：演练 + 迭代预告

---

## 1. 明确愿望（Aspiration）

先把用户真正想要的**结果或愿望**说清楚——是抽象的方向（"睡得更好""成为持续输出的写作者"），不是具体行为。
- 问："你最想在生活里多出来的，是什么样的状态或结果？"
- 区分**愿望**（aspiration，抽象，如"更健康"）和**结果**（outcome，可测量，如"瘦 5 公斤"）。两者都可以当起点，但都还不是行为。
- 关键提醒：**愿望本身做不到，能做的只有行为**。所以下一步是把愿望翻译成一堆候选行为。

## 2. 探索行为群（Magic Wand）

用"魔法棒"发散，帮用户列出**一大群**能服务这个愿望的具体行为，别急着评判好坏：
- 问："如果有根魔法棒，挥一下你就能毫不费力地做到任何事——哪些**具体行为**能帮你实现这个愿望？尽量多想，先不管做不做得到。"
- 帮他从几个角度凑数量：
  - **一次性的行为**（如"买一双好跑鞋"）
  - **可重复的新习惯**（如"每天走 20 分钟"）
  - **想停掉的行为**（如"少喝一杯奶茶"）
- 目标是凑出**十几二十个**候选，形成"行为群"。数量比质量重要，发散阶段不筛选。

## 3. 焦点地图：筛出黄金行为（Focus Mapping）

把行为群放进一张二维图，找出**黄金行为**（既有用、又你真做得到的那几个）：

```
影响力 高
  ↑   |  （影响大但很难做到，    |  ★ 黄金行为 ★
      |    先放一边）            |  影响大 + 你真做得到
      |------------------------|------------------------
      |  （没用又难，直接划掉）  |  （容易但没用，别浪费在这）
  低  +------------------------+----------------------→
         你实际能做到/愿意做的程度  高
```

操作：
- **纵轴 = 这个行为对实现愿望的影响力**：一个个问用户"这件事如果做到了，对你的愿望帮助大不大？"
- **横轴 = 你实际能做到、且愿意做的程度**（注意是"你"，不是"一般人"——要诚实）。
- 落在**右上角**的就是**黄金行为**：影响大 + 你真能做到。挑出 1-3 个，作为接下来要设计的习惯。
- 这一步是整个流程的灵魂：**不是逼自己去做最难最有用的，而是找到对你既有用又做得到的那个甜点区。**

## 4. 从微小起步（Start Tiny）

把每个黄金行为缩到**极小**——小到几乎不需要动机、30 秒内能做完。两种缩法：
- **起步步骤（Starter Step）**：只做整件事的第一个最小动作（"想跑步" → "穿上跑鞋"；"想冥想" → "做一次深呼吸"）。
- **缩小规模（Scale Back）**：把完整行为砍到迷你版（"读 30 分钟" → "读一段"；"做 20 个俯卧撑" → "做 2 个"）。
- 检验标准：用户听到都觉得"这也太简单了吧" = 对了。**容易，是为了让它在动机最低的日子也照样发生**；做多了是奖励，不是要求。

## 5. 找锚点：After-I-will 配方（Anchor）

给微行为找一个**已经稳定发生的日常动作**当提示，绑成配方：

> **在我〔锚点：现有习惯〕之后，我会〔微行为〕。**
> （例：在我倒完早上第一杯咖啡之后，我会做 2 个俯卧撑。）

挑锚点的三条匹配原则（帮用户挑得准）：
- **地点匹配**：锚点和新行为最好在同一个地方能完成。
- **频率匹配**：锚点发生的频率 = 你想要新习惯的频率（想每天做，就锚在每天必做的事上）。
- **主题/节奏匹配**：锚点结束时的状态，顺得上新行为（刚坐到书桌前 → 适合接"写一句话"）。

让用户把配方**用自己的话说出来**，越具体越好（哪杯咖啡、哪张椅子）。

## 6. 设计庆祝（Celebrate）

这是福格区别于其他习惯理论的核心：**让习惯扎根的是情绪，不是次数。** 做完微行为的**那一瞬间**，立刻给自己制造一股真实的正面情绪（福格叫 "Shine / 闪耀"）。
- 问用户："做完那一下，你做点什么会让自己真的感到一阵好心情？"（握拳说"漂亮！"、哼一句歌、想象观众鼓掌、对自己笑一下……）
- 必须是**他自己觉得真的爽**的，不是表演。
- 时机关键：**就在行为完成的当下**，不是事后。
- 想深入（庆祝菜单、找到属于你的庆祝方式）→ 读 `celebrate.md`。

## 7. 收尾：演练 + 迭代预告

- 让用户**当场把整套配方演练一遍**：触发锚点 → 做微行为 → 立刻庆祝。哪怕只是脑子里走一遍。
- 预告迭代心态：**先让微习惯稳定地"长在"生活里，它自己会慢慢长大**（想多做时随时加量，但底线永远是那个微版本）。坚持不下来不丢人，是配方要调——回诊断模式查 B=MAP 哪一环松了。
- 别忘 SKILL.md 的结尾动作：问用户要不要把这份习惯配方整理成一份小结。
