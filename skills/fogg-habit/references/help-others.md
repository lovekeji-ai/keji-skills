# 帮别人改变模式：把行为设计用在家人/团队

福格全书后半的延伸：行为设计不只能改自己，也能帮**别人**改变——伴侣、孩子、团队。核心原则一句话：**你没法直接改变别人的行为，但你能帮他们成功。**

---

## 1. 黄金法则：帮他成功，别施压

- 福格："帮助别人改变 = 帮他们感到成功（feel successful）"，而不是说教、施压、讲道理。
- **动机施压几乎总是失败**：催、唠叨、激将，短期或许有效，长期破坏关系也破坏习惯。
- 你能动的，是帮对方**把行为变简单**、**装好提示**、**在他做到时一起庆祝**。

## 2. 三步：先找出真正想要的行为

帮别人前先想清楚：
1. **明确目标行为**：你希望对方多出现的、**具体的**行为是什么（不是"希望孩子更自觉"，而是"放学回家先把书包放进固定位置"）。
2. **确认这是不是对方也想要的**：如果对方根本不想要，你能做的很有限，别硬来——换一个双方都认的行为。
3. **用焦点地图思路**：在"对目标有用 × 对方真做得到"里挑黄金行为（参考 `design.md` 第 3 步）。

## 3. 帮对方降低难度 + 设好提示

- **降低难度**：用能力五要素（时间/金钱/体力/脑力/打破惯例）看对方卡在哪，帮他把行为缩小、铺好路（孩子要收拾玩具 → 准备一个一脚就能踢进去的大筐）。
- **设好提示**：帮对方把新行为锚在他已有的日常动作后面，而不是靠你提醒（你一提醒就变成催促，伤关系）。

## 4. 制造庆祝与正反馈

- 对方做到的当下，**及时、真诚地给正反馈**，帮他产生"我做到了"的好情绪——这就是替对方制造 Shine。
- 对孩子尤其有效：真实的鼓励 > 物质奖励。注意是**针对行为**的具体肯定（"你自己把书包放好了，真利索"），不是空泛夸"你真棒"。

## 5. 对不同对象的差异

- **孩子**：行为要极小、提示要直观、庆祝要夸张而真诚；把养习惯做成游戏。
- **伴侣**：重点是**别施压、别当对方的监工**。你负责把环境和提示弄顺，让对方更容易做到；做到了一起高兴。关系优先于习惯本身。
- **团队**：聚焦"让正确的行为更容易发生"——改流程、改默认项、减步骤，比开会强调"大家要注意"有效得多。

## 6. 收尾

- 给用户一个具体方案：**目标行为 + 怎么帮对方变简单 + 提示锚在哪 + 你怎么给正反馈**。
- 提醒：盯着"帮他成功、别施压"这条底线，效果不好就调环境，别调对方的态度。
- 别忘 SKILL.md 结尾：问要不要把这个方案整理成一份小结。
