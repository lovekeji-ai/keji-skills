# 诊断排错模式：为什么没坚持下来

用于用户已经有一个想坚持但断掉/没养成的习惯，想知道**原因 + 怎么改**。

核心方法：拿 B=MAP 反向排查。福格的铁律是**按顺序查，不要乱跳**——先提示，再能力，最后才动机。绝大多数"坚持不下来"的真相在前两项，不在意志力。

## 目录
1. 先还原现场
2. 第一步查提示（Prompt）
3. 第二步查能力（Ability）
4. 第三步才查动机（Motivation）
5. 给出诊断结论 + 重新设计

---

## 1. 先还原现场（问 1-2 个问题）

先让用户把这个习惯讲具体，别停留在"我想多运动"这种模糊愿望：
- "你原本打算做的具体行为是什么？在什么时候做？"
- "最近一次做到 / 最近一次断掉，分别是什么情况？"

把它落成一个具体行为（"晚饭后跑步 30 分钟"），后面才好排查。

## 2. 第一步查提示（Prompt）——最常见的元凶

没有提示，行为永远不会发生。先问：
- "你靠什么来触发它？是想起来就做，还是绑在某个固定动作后面？"

判断：
- **根本没有提示** → 只是"想着要做"，靠记忆 = 没提示。这是头号死因。
- **提示是手机闹钟/App 推送** → 容易被忽略、被划掉，是弱提示。
- **提示和行为脱节** → 提示出现的时间/地点，根本没法马上做那件事。

改法预告（细节在行为设计模式）：把新行为**锚定在一个已经稳定发生的日常动作之后**——"在我〔已有习惯〕之后，我会〔新行为〕"。锚点要在**地点、频率、主题**上和新行为匹配。

## 3. 第二步查能力（Ability）——行为是不是太难

福格：**让行为更容易，几乎总比提升动机更有效、更持久。**问用户这件事难在哪，逐一过"能力五要素"——哪一项卡住就攻哪一项：

| 能力要素 | 卡点表现 | 松绑方向 |
|---|---|---|
| **时间** | "没时间做" | 砍到几分钟内能完成 |
| **金钱** | "要花钱/办卡才能做" | 找零成本版本 |
| **体力** | "做起来太累" | 降强度、减量 |
| **脑力** | "要动脑、要做决定、要查资料" | 提前准备好、去掉决策 |
| **日常惯例** | "得打乱我原本的安排" | 嵌进现有流程，别另起炉灶 |

关键追问："如果把这件事砍到**小到不可能失败**，它会是什么样子？"（跑步 30 分钟 → 换上跑鞋出门即可；读书 → 读一段）。用户往往把行为定得太大，这一步就是帮他**缩小规模**。

## 4. 第三步才查动机（Motivation）——而且通常结论是"别靠它"

只有提示和能力都没问题时，才轮到动机。要点：
- **动机是波动的**：今天高、明天低。如果一个习惯**只有在动机高时才做得到，说明它被设计得太难了** → 回到第 2 步把它变简单，而不是想办法长期维持高动机。
- 动机分三层（核心动机）：愉悦/痛苦、希望/恐惧、社会接纳/排斥。可以问"你为什么想要这个习惯"来确认愿望真不真，但**不要把方案建立在'我要更自律'上**。
- 如果发现用户其实**没那么想要**这个结果 → 诚实点破，也许这根本不是他的黄金行为，值得换一个（接行为设计模式重新选）。

## 5. 给出诊断结论 + 重新设计

收尾时给用户一个清楚的结论，结构：
1. **真正的卡点**：是提示缺失 / 行为太难 / 动机不可靠（按 B=MAP 定位，通常 1-2 项）。
2. **不是你的错**：强调这是设计问题，不是意志力问题——别让用户自责。
3. **下一步怎么改**：针对卡点给具体改法。如果需要从头重做，就自然过渡到**行为设计模式**（读 `design.md`），把这个习惯按"微行为 + 锚点 + 庆祝"重新设计一遍。

别忘了 SKILL.md 里的结尾动作：问用户要不要把诊断结论整理成一份小结。
